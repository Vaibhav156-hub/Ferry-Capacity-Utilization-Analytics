# 🚢 Ferry Capacity Utilization & Operational Efficiency Analytics System

## 📌 Project Overview

This project analyzes **Toronto Island Ferry Ticket Activity (2015–2025)** to evaluate ferry capacity utilization, identify congestion-prone and idle-capacity periods, and provide data-driven operational insights.

The project includes data preprocessing, feature engineering, exploratory data analysis (EDA), KPI analysis, and an interactive Streamlit dashboard for visualization.

---

## 🎯 Objectives

- Analyze ferry capacity utilization
- Identify congestion and idle-capacity periods
- Evaluate operational efficiency
- Compare seasonal and time-based activity
- Build an interactive Streamlit dashboard
- Generate business insights and recommendations

## 🚀 How to Run

Clone the repository:

```bash
git clone https://github.com/Vaibhav156-hub/Ferry-Capacity-Utilization-Analytics.git
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Run the Streamlit application:

```bash
streamlit run app.py
```

## 📈 Key Performance Indicators (KPIs)

- Total Ticket Sales
- Total Ticket Redemptions
- Total Activity Load
- Average Operational Load Index (OLI)
- Congestion Percentage
- Idle Capacity Percentage
- Peak Activity Hour
- Peak Activity Day

---

## 💡 Business Insights

- Peak passenger activity occurs during specific hours of the day.
- Seasonal demand significantly affects ferry operations.
- Congestion is concentrated in a limited number of operational intervals.
- Idle-capacity periods highlight opportunities for resource optimization.
- Historical ticket activity supports data-driven scheduling decisions.
